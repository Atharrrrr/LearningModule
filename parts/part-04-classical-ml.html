Part IV · Chapter 4.1

Linear Regression

Chapter 1.3 fit a line through exam scores and called it "curve fitting." That technique has a real, official name — and this chapter extends it to combine several clues at once.

🏘️ Frame of reference

A real estate appraiser doesn't estimate a house's value from just one clue. They weigh square footage, number of bedrooms, and age together — each contributing its own share to the final number. Chapter 1.3's line only ever used one clue (hours studied). This chapter's whole upgrade is combining several clues into a single weighted prediction — otherwise, it's the exact same idea.

The technique already has a name

Linear regression is exactly Chapter 1.3's score = m × hours + b, officially named. With more than one feature, it becomes:

prediction = w₁×feature₁ + w₂×feature₂ + ... + bias

Weights: one knob per feature

Each feature gets its own weight — the general name for what Chapter 1.3 called m, now that there's more than one to keep track of. A larger-magnitude weight means that feature swings the prediction more; a weight near zero means it barely matters.

Worked example: pricing a house with two clues

Reusing Chapter 3.4's housing features, suppose price = 150 × sqft − 2000 × age + 50000. For a 2,000 sqft house that's 10 years old:

price = 150×2000 − 2000×10 + 50000 = 300,000 − 20,000 + 50,000 = $330,000

Part IV · Chapter 4.2

Logistic Regression

Despite the name, this isn't for predicting a number — it's for predicting a category, by squashing Chapter 4.1's weighted sum into a probability that can never leave 0 to 1.

🎚️ Frame of reference

Picture a confidence dial that physically cannot turn past its end stops — no matter how hard you push it, it curves and flattens out near 0% and 100% instead of shooting past them. Linear regression's raw weighted sum can output any number at all — even far below 0 or above 1, which makes no sense as a probability. Logistic regression bolts that confidence dial onto the end of the calculation, guaranteeing a sensible answer every time.

Squashing a number into a probability

Logistic regression is for classification — predicting a category, not a number. It computes the exact same weighted sum as Chapter 4.1's linear regression, then passes that raw score through the sigmoid function, which squashes any real number into a value between 0 and 1. That output is read as a probability: "92% chance this is spam." A threshold — commonly 0.5 — turns the probability into a final yes/no decision.

Worked example: scoring an email

Say a spam filter's weighted sum for one email comes out to 2.5 (positive score = spam-leaning). The sigmoid function turns that into sigmoid(2.5) ≈ 0.92 — a 92% chance of spam, comfortably over the 0.5 threshold, so it's classified spam. A different email scoring -1.5 gives sigmoid(-1.5) ≈ 0.18 — 18%, classified not spam.

  • Logistic regression takes linear regression's weighted sum and squashes it through the sigmoid function into a 0-1 probability.
  • Despite its name, it's used for classification, not predicting a continuous number.
  • A threshold (commonly 0.5) turns the probability into a final decision.
  • The sigmoid function guarantees the output can never go below 0 or above 1, no matter how extreme the raw score.
