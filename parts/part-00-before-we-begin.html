How This Book Works

Before any math, code, or models — a quick tour of how this book is built, so the format never gets in the way of the ideas.

🎮 Frame of reference

Think of this book like a video game with a main quest and side quests. The main quest is a straight line from "knows nothing about AI" to "job-ready ML engineer" — you can finish the whole game by only playing those missions. Side quests are optional: they go deeper into a topic, reward you with extra understanding, and you can play them whenever you want, in any order, or skip them entirely without getting stuck.

Parts, chapters, and two kinds of chapter

The book is organized into 16 Parts (Part 0 through Part XV), and each Part is broken into numbered chapters like the one you're reading now — 0.1. Every chapter carries one of two tags, taken straight from CURRICULUM.md:

  • [C] Core — the main quest. Read every Core chapter in order and you'll reach job-ready, no exceptions and no gaps.
  • [D] Deep-dive — a side quest. Extra depth on one topic (history of AI, scaling laws, distributed training, and so on). Skip it, read it later, or read it now — the Core path never depends on it.

You'll see that word — algorithm — again soon. That box above it is the book's most important mechanic: the first time any jargon word shows up anywhere in the book, it gets a short definition box right where it appears. After that first time, later mentions link back to the original definition instead of repeating it, so the book never explains the same thing to you twice.

The toolbox you'll see in every chapter

Besides jargon boxes, four more things show up throughout:

  • Interactive demos — small, real, click-around visuals built with SVG and JavaScript, meant to build intuition by letting you play with an idea rather than just read about it.
  • Python code blocks — labeled "Python — run in your own environment." These illustrate what real code looks like but don't execute inside the page; copy them into your own Python setup if you want to run them.
  • Quizzes — two or three click-to-reveal questions at the end of every chapter. Wrong answers cost nothing; you get instant feedback and can keep trying.
  • Progress tracking — the sidebar on the left fills in a dot for every chapter you've scrolled through, saved locally in your browser (not sent anywhere) so it's still there next time you open the book.

Quick check
1. If you only read every [C] Core chapter and skip every [D] Deep-dive chapter, what happens?
You'll get stuck, because some Core chapters assume you read the Deep-dive chapters first.
You'll still reach job-ready — the Core path is complete on its own.
You'll miss chapters about topics unrelated to AI/ML.
2. A jargon word appears for the second time in a later chapter. What should happen?
It gets a brand-new definition box again, in case you forgot.
It links back to where the term was first defined, instead of redefining it.
It's left undefined, since jargon is only explained once per Part.
3. Where is your chapter-reading progress actually stored?
On the book's server, tied to your account.
Locally, in your own browser's storage.
It's emailed to you at the end of each chapter.
