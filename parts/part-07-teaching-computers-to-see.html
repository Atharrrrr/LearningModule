Part VII · Chapter 7.1

Images Are Just Grids of Numbers

Everything in this Part — recognizing cats, detecting cars, matching keypoints between photos — is built on one simple fact: a computer never actually sees a picture. It sees a grid of numbers, and nothing else.

🏟️ Frame of reference

A stadium card display, where thousands of fans each hold up a single colored card on cue. From high in the stands, it resolves into a giant picture. Walk down onto the field, though, and the picture disappears — all you see is one person holding one card with one specific color. The picture was never actually "in" the display; it only ever existed as many individually addressable colors, arranged in a grid. A digital image works exactly the same way.

The pixel: one number (or three)

A pixel is one cell in that grid. In a grayscale image, each pixel is a single number from 0 (black) to 255 (white), with everything in between a shade of gray. Stack pixels into rows and columns and you get exactly Chapter 2.3's matrix: one row per image row, one column per image column, one number per pixel.

With the numbers hidden, this reads as a picture — a plus sign. Reveal them, and it's obviously just a grid of two numbers, 230 and 60, arranged in a pattern. Nothing about the image "knows" it's a plus sign; that's a pattern a human — or, as the rest of this Part shows, a trained model — recognizes in the numbers.

Color: three grids stacked together

A color image adds two more grids. Each channel — red, green, and blue — is its own grayscale grid, exactly like the plus sign above. Combine the three channels' values at each position, and you get a color. This is precisely Chapter 2.11's tensor: height × width × 3 stacked matrices, one per channel.

✓ What you can now explain to someone else
  • A pixel is one cell in an image's grid, holding one number (grayscale) or three (color).
  • A grayscale image is exactly Chapter 2.3's matrix: one number per pixel, arranged in rows and columns.
  • A color image is three grayscale grids (red, green, blue channels) stacked together — exactly Chapter 2.11's tensor.
  • A computer never "sees" a picture directly; every operation in this Part works on these numbers.

Part VII · Chapter 7.2

Convolutional Neural Networks

Feed a whole image into Chapter 6.2's fully-connected layers, and two problems show up immediately: an enormous number of weights, and a network with no built-in sense that nearby pixels are related. Convolution fixes both, using an idea that's already familiar.

🔍 Frame of reference

Scanning a whole page of text for the letter "e," using a small magnifying glass. You don't need a different expert stationed at every position on the page — the same small "does this look like an e?" check gets slid across every patch of the page, one small window at a time. The exact same detector, reused everywhere, rather than one giant detector custom-built for the entire page at once.

The convolution: a dot product, slid across an image

A kernel (filter) is a small grid of weights — commonly 3×3. At every position on the image, it multiplies its weights against the pixels underneath it and sums the results: exactly Chapter 2.4's dot product, computed between the kernel and whatever small patch of the image it's currently sitting over. Slide it across every position, and each dot product becomes one value in a feature map — a convolutional neural network (CNN) is built from layers of exactly this operation.

The vertical-edge kernel lights up wherever brightness changes left-to-right — the plus sign's left and right edges — and goes completely flat (zero) down the middle and outer columns, where nothing changes horizontally. The horizontal-edge kernel does the mirror opposite, lighting up the plus sign's top and bottom edges. Same operation, different weights, different feature detected — this is the entire idea a CNN's early layers learn automatically, rather than needing a human to hand-pick "detect vertical edges" as a rule.

Why not just use a fully-connected layer?

A fully-connected layer (Chapter 6.2) mapping this 8×8 = 64-pixel image to a 6×6 = 36-value output would need a separate weight for every input-output pair: 64 × 36 = 2,304 weights. The convolution above produced that same 36-value output using just 9 weights — the kernel — reused at all 36 positions. That weight sharing is the whole efficiency story: a pattern worth detecting in one corner of an image is usually just as worth detecting anywhere else, so the same small detector gets reused everywhere instead of learned separately, over and over, per position.

Stack several convolutional layers and something else emerges: early layers detect simple things like edges (exactly what's shown above); layers built on top of those combine edges into corners and textures; layers built on top of those combine textures into parts and shapes. Chapter 7.5 tours the famous architectures built this way.

✓ What you can now explain to someone else
  • A kernel is a small grid of weights that computes a dot product with the image patch underneath it, at every position.
  • Sliding a kernel across an image produces a feature map, showing where that kernel's pattern is strongly present.
  • The same kernel is reused at every position (weight sharing), using far fewer parameters than a fully-connected layer over the same input.
  • Stacked convolutional layers build up from simple features (edges) to increasingly complex ones (textures, parts, shapes).
